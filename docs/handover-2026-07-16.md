# 핸드오버 문서 — 대회 수정 버그 수정 + 대진표(토너먼트 브라켓) 기능 신규 (2026-07-16)

## 변경 요약

지난 세션(`docs/handover-2026-07-12.md`) 이후 진행된 작업. (1) 관리자에서 대회를 수정해도 반영 안 되던 캐싱 버그를 고치고 수정폼을 대회 상세 페이지와 1:1로 맞췄고, (2) 대회장 썸네일 아이콘을 검도와 안 맞는 이모지에서 죽도 아이콘으로 바꿨고, (3) sportsprism.net 스타일의 전체 대진표(토너먼트 브라켓) 기능을 스키마부터 렌더링, 관리자 입력 UI까지 새로 만들었습니다. 코드는 `9e83c1d`까지 **커밋+push 완료**. SQL 마이그레이션 파일 2개에 문법 수정이 남아있는데(아래 참고) 아직 커밋 안 됨.

### 대진표 기능 상세 (`9e83c1d`)

- `bracket_matches` 테이블 신규 — 조(A/B/결승)·라운드·순번 기반 이진트리 구조, 선수는 `competition_participants`에 FK 연결하지 않고 이름/소속 자유 텍스트 + "우리 클럽" 불리언으로 표시만 강조
- 대회 상세 페이지에 "대진표" 탭 추가 — CSS Grid만으로 A조/B조 트리와 꺾쇠 연결선을 그림 (신규 npm 패키지 설치 없음), 1~3위 포디움은 결승/준결승 결과에서 자동 계산, 기존 "대진표" 파일(사진/PDF 업로드)도 같은 탭에 노출
- 관리자 화면(대회 관리 → 대회 목록 → 대회 펼치기)에 매치 단위 입력/수정/삭제 UI 추가
- **`bracket_matches`는 `competitions` 조회에 JOIN하지 않고 별도 쿼리로 분리**해뒀음 — 처음엔 JOIN으로 넣었다가, 테이블이 없는 상태로 배포되면 대진표와 무관한 모든 대회 상세/목록 페이지가 통째로 깨지는 것을 발견해서 배포 직전에 고침

## 아직 커밋 안 된 변경사항

### 수정된 기존 파일

| 파일 | 변경 내용 | 이유 |
|------|----------|------|
| `docs/migration-bracket-matches-2026.sql` | `create policy if not exists ...` → `drop policy if exists ...; create policy ...`로 변경 | Postgres에서 `create policy if not exists` 문법 자체가 유효하지 않아 Supabase SQL Editor 실행이 실패함. 사용자가 직접 고쳐서 실행 성공시킴 |
| `docs/migration-competition-2026.sql` | 위와 동일한 정책 문법을 전체 8개 정책에 대해 동일하게 수정 | 첫 마이그레이션(가장 오래된 파일)에도 같은 무효 문법이 있었던 것을 이번에 함께 발견/수정 |

두 파일 다 **DB에는 이미 반영 완료**(사용자가 수정된 SQL로 재실행함) — 재실행은 필요 없고, 파일 내용을 커밋만 하면 됨.

## 배포 전 필수 작업

1. **위 SQL 파일 2개 커밋 여부 확인** — 아직 사용자에게 커밋 지시를 받지 않은 상태.
2. **대진표 매치 데이터는 아직 하나도 입력 안 됨** — `bracket_matches` 테이블은 만들어졌지만 비어있어서, 실제 배포 화면에서 "대진표" 탭이 아직 어디에도 안 보이는 게 정상. 사용자가 관리자 화면에서 최소 1개 대회에 매치를 입력해봐야 기능이 눈에 보임 (직전 대화에서 테스트 방법 안내함, 아직 결과 확인 전).

## 기능 사용 흐름 (신규 — 대진표)

1. `/admin` → 대회 관리 → 대회 목록 → 대상 대회 펼치기 → 맨 아래 "🏆 대진표 매치" → "+ 매치 추가"
2. 최소 예시(4인 대진, 3매치): 부문명 동일하게 맞춰서 A조 R1-1 / B조 R1-1 / 결승 R1-1 순서로 입력, 각 매치에 승자 지정
3. 대회 상세 페이지(`/competition/[id]`)로 가면 매치가 1개 이상 있어야 "대진표" 탭이 노출되고, 결승/준결승 결과가 채워지면 1~3위 포디움이 자동으로 뜸
4. 대진표 사진/PDF를 파일로 올리고 싶으면 기존 파일 업로드에서 유형 "대진표" 선택 — 구조화 매치 데이터와 별개로 같은 탭 하단에 함께 표시됨

## 알려진 제약 / 유의사항

- **대진표 side당 인원은 2의 거듭제곱만 지원** (4/8/16명 등), 부전승은 1라운드에서만 지원 — 2라운드 이후의 변칙 부전승/불균등 대진은 미구현(후속 과제로 남김).
- **division(부문)은 자유 텍스트** — 같은 대진표인데 "남자노년부"/"남자 노년부"처럼 오타나면 트리가 두 개로 갈라져 보임. 관리자 폼에 기존 입력값 datalist 제안은 있지만 강제하진 않음.
- **`bracket_matches`는 절대 `competitions` 조회에 JOIN하지 말 것** — 별도 쿼리로 유지해야 이 테이블에 문제가 생겨도 대회 목록/상세 전체가 죽지 않음. 향후 이 영역 코드 수정 시 이 패턴 유지 필요.
- **Postgres에서 `create policy if not exists` 문법은 무효** — 이번에 처음 발견됨. 앞으로 새 마이그레이션 SQL을 작성할 때는 반드시 `drop policy if exists "이름" on 테이블; create policy "이름" on 테이블 ...;` 패턴을 사용할 것 (기존 `docs/migration-*.sql` 전부 이 패턴으로 통일됨).
