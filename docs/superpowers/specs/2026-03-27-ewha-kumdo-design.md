# EWHA Kumdo — 웹앱 설계 문서

**작성일:** 2026-03-27
**상태:** 승인 대기

---

## Context

검도 훈련 영상을 유튜브에 비공개(링크 공유)로 업로드하고 있으나, 매번 참가자에게 링크를 개별 전달해야 하는 불편함이 있다. 2명의 업로더가 각각 다른 유튜브 계정에서 전면/후면 앵글로 촬영한 영상을 올리고 있어 열람이 더 어렵다.

**목적:** 유튜브 링크를 중앙에서 관리하고, 날짜·참가자·앵글 기준으로 검색·분류할 수 있는 웹앱을 구축한다.

## 데이터 모델 (Supabase)

### `videos` 테이블

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `id` | uuid | — | 자동 생성 (PK) |
| `youtube_url` | text | ✅ | 유튜브 영상 링크 |
| `title` | text | ✅ | 영상 제목/설명 |
| `date` | date | ✅ | 촬영 날짜 (YYYY-MM-DD) |
| `angle` | text | ✅ | `"전면"`, `"후면"`, `"기타"` 중 선택 |
| `participants` | text[] | ⬜ | 참가자 이름 배열 e.g. `["홍길동", "이순신"]` (선택) |
| `topic` | text | ⬜ | 주제 자유 입력 (선택) |
| `uploader` | text | ⬜ | 등록자 이름 (선택) |
| `created_at` | timestamptz | — | 자동 생성 |

**날짜 표시 형식:** `date` 필드에서 요일을 프론트엔드에서 계산하여 `2025. 03. 20 (목)` 형식으로 출력.
**페어 영상 매칭:** 별도 필드 없음. `date`가 동일한 영상을 쿼리로 묶어 표시.

## 레이아웃 구조

### 데스크톱
- **헤더 (고정):** ☰ 버튼 | EWHA Kumdo 로고 | 검색창 | `+ 영상 등록` 버튼
- **사이드바 (접기/펼치기):** 홈 / 캘린더 보기 / 주제별 보기 / 참가자별 보기
- **필터 칩 바:** 참가자 태그, 앵글, 날짜 등 동적 필터
- **영상 그리드:** 4:3 가로형 카드, `auto-fill` 반응형 열

### 모바일
- 사이드바 숨김 → **하단 탭 네비게이션** (홈/캘린더/주제별/참가자별)
- ☰ 메뉴는 오버레이 슬라이드로 보조 제공
- 필터 칩: 가로 스크롤
- 그리드: 2열 고정

## 관리자 페이지 (`/admin`)

1. **비밀번호 입력 화면** — 서버에서 `ADMIN_PASSWORD` 환경변수와 대조
2. **인증 통과 시 대시보드 진입**
   - **단건 등록 폼:** YouTube URL → 날짜(date picker) → 앵글 선택 → 참가자 입력(태그 형식) → 제목 → 주제(선택)
   - **CSV 대량 업로드:** 컬럼 순서 `youtube_url, date, angle, participants, title, topic`
   - **영상 목록:** 등록된 영상 카드 + 수정/삭제 버튼

비밀번호는 Vercel 환경변수로 관리 — 코드/클라이언트에 노출 없음.

## 캘린더 페이지 (`/calendar`)

월간 달력 형태. 영상이 존재하는 날짜에 앵글별 배지를 표시.

- **배지 색상:** 전면 = Ewha Green(`#00462A`) / 후면 = 진회색(`#374151`) / 기타 = Ewha Grey
- **배지 클릭:** 해당 영상 상세 페이지(`/video/[id]`)로 직접 이동
- **오늘 날짜:** 초록 원 표시
- **이전/다음 달 날짜:** 흐리게(grey) 표시
- **월 이동:** `‹` `›` 버튼으로 월 전환
- **범례:** 캘린더 상단에 전면/후면/기타 색상 안내 표시

예시 (3월 15일에 전면·후면 영상이 각각 1개인 경우):
```
15
[ 전면 ]  → 클릭 시 해당 전면 영상으로 이동
[ 후면 ]  → 클릭 시 해당 후면 영상으로 이동
```

같은 날짜·같은 앵글 영상이 2개 이상인 경우: 배지를 개수만큼 반복 표시 (예: 전면 2개 → `[ 전면 ][ 전면 ]`).

## 배포 및 비용

| 서비스 | 플랜 | 무료 한도 | 예상 사용량 |
|--------|------|-----------|-------------|
| Vercel | Hobby (무료) | 대역폭 100GB/월 | ~수십 MB/월 |
| Supabase | Free | DB 500MB | ~1MB 미만 |

- 비용 발생 시점: 영상 수천 개 + 일 방문자 수천 명 수준 (현재의 수백 배)
- Supabase 주의: 1주 이상 미접속 시 DB 일시 정지 → 첫 요청 시 자동 재가동 (1~2초)
- 이관 용이성: 표준 Next.js + PostgreSQL — Netlify/Railway 등으로 언제든 이전 가능

## 초기 데이터 마이그레이션

기존 영상 약 100개는 CSV 파일로 정리 후 `/admin` 페이지의 CSV 업로드 기능으로 일괄 등록.

CSV 예시:
```
youtube_url,date,angle,participants,title,topic
https://youtu.be/xxx,2025-01-10,전면,"홍길동,이순신",검도 훈련,기본기
https://youtu.be/yyy,2025-01-10,후면,"홍길동,이순신",검도 훈련,
```

## 검증 방법

1. Vercel 배포 후 공개 URL로 영상 그리드 정상 로드 확인
2. 참가자 태그 클릭 → 해당 참가자 영상만 필터링 확인
3. `/admin` 비밀번호 인증 → 영상 등록 → 홈에 즉시 반영 확인
4. CSV 업로드 → 다수 영상 일괄 등록 확인
5. 캘린더 페이지에서 날짜별 영상 분포 확인
6. 모바일 브라우저에서 하단 탭 네비 및 2열 그리드 정상 동작 확인
