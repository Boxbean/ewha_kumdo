## 배포 전 필수 작업 (다음 세션에서 이어서)

### 1. Supabase SQL 실행 (미완료)
`docs/migration-venue-facilities-2026.sql`을 Supabase Dashboard > SQL Editor에서 실행 — 실행 전까지 대회장 관리 폼에서 "주변식당 및 편의시설"/"경기장 특이사항" 저장 시 API가 컬럼 없음 오류를 낼 수 있음.

### 2. 로컬 환경변수 설정 (미완료)
로컬에 Node.js/npm 설치 완료(`winget install OpenJS.NodeJS.LTS`). `npm install` 완료. 다만 `.env.local`이 비어있어 로컬 `npm run dev` 시 Supabase 연결 불가 — Vercel 대시보드의 값을 `.env.local`에 채워야 함(임시로 `ENV_TEMPLATE.txt`에 채우면 `.env.local`로 옮겨줄 예정, 아직 미완료).

### 3. 로컬 확인 후 커밋/푸시
사용자가 `npm run dev`로 직접 화면 확인 후, 문제 없으면 위 "아직 커밋 안 된 변경사항"을 커밋 + push하기로 함 (사용자 명시적 지시 대기 중).

## 기능 사용 흐름 (신규)

### 대회별 / 경기장별 보기
- `/competition` = "대회별" 뷰: 사회인대회/서울컵대회/서울시 회장기대회/춘·추계 서울시 대학연맹전/금천구청장기 대회/대선기대회/도봉구청장기 대회 7개 카드가 데이터 유무와 관계없이 항상 노출, 각 카드는 가장 최근 개최일·장소를 보여줌
- 카드 클릭 → `/competition/series/[key]` — 해당 시리즈의 연도별 개최 이력 전체
- `/competition/venues` = "경기장별" 뷰: 등록된 대회장 목록 → 클릭 시 `/venue/[id]`에서 해당 대회장의 개최 이력 확인

### 대회 상세 페이지
- 대회장 정보(주소·주차·코트수·바닥·규모·교통·주변식당·특이사항)가 탭 없이 상단에 항상 노출
- 특이사항(`notes`)도 상단에 항상 노출
- 팸플릿 파일이 있으면 바로가기 칩으로 상단에 노출

## 알려진 제약 / 유의사항

- 춘/추계 대학연맹전 병합은 **화면 표시 레이어에서만** 이뤄짐 — 관리자가 대회를 등록할 땐 여전히 "서울시 춘계 대학연맹전"/"서울시 추계 대학연맹전"을 각각 정확히 선택해야 시리즈 페이지에 정상적으로 묶여서 나타남.
- `nearby_info`/`notes`는 대회장(venue) 레벨 필드 — 같은 대회장에서 열린 모든 대회가 값을 공유함(대회 인스턴스별로 다르게 저장되지 않음).
- 이번 세션에서 로컬에 Node.js를 처음 설치함(이전 세션까지는 미설치 상태로 Vercel 빌드만으로 검증했음).
